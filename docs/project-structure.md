# Project Structure

## Root

```
rik/
├── CLAUDE.md                    # AI agent instructions
├── docker-compose.yml           # All services
├── .env.example                 # Template for environment variables
├── docs/                        # Architecture & design docs (you are here)
├── app/                         # TanStack Start app + worker
├── packages/
│   └── cli/                     # rik CLI — talk to Rik from terminal
├── config/                      # External service configs
└── data/                        # Persistent data (git-ignored)
```

## App (`app/`)

The TanStack Start application — serves the frontend, runs server functions, and shares code with the worker.

```
app/
├── Dockerfile                   # App container build
├── package.json
├── tsconfig.json
├── app.config.ts                # TanStack Start config
├── drizzle.config.ts            # Drizzle Kit config
├── src/
│   ├── routes/                  # TanStack Router — file-based routes
│   │   ├── __root.tsx           # Root layout (shell, sidebar, providers)
│   │   ├── index.tsx            # Dashboard
│   │   ├── mail/
│   │   │   ├── index.tsx        # Email inbox
│   │   │   └── $emailId.tsx     # Single email view
│   │   ├── tasks/
│   │   │   ├── index.tsx        # Task list
│   │   │   └── $taskId.tsx      # Single task view
│   │   └── invoices/
│   │       ├── index.tsx        # Invoice list
│   │       └── $invoiceId.tsx   # Invoice detail
│   │
│   ├── components/              # UI components
│   │   ├── ui/                  # shadcn/ui components (auto-generated)
│   │   ├── dashboard/           # Dashboard-specific components
│   │   │   ├── agenda-widget.tsx
│   │   │   ├── tasks-widget.tsx
│   │   │   └── mail-widget.tsx
│   │   ├── mail/                # Email-specific components
│   │   ├── tasks/               # Task-specific components
│   │   └── invoices/            # Invoice-specific components
│   │
│   ├── server/                  # Server-side code (never sent to client)
│   │   ├── functions/           # TanStack Start server functions
│   │   │   ├── mail.ts          # Email CRUD + search
│   │   │   ├── tasks.ts         # Task CRUD
│   │   │   ├── invoices.ts      # Invoice CRUD
│   │   │   ├── agenda.ts        # Calendar events
│   │   │   ├── chat.ts          # Chat endpoint — LLM + tool use
│   │   │   ├── webhooks.ts      # Stalwart webhook receiver
│   │   │   └── jobs.ts          # Enqueue background jobs
│   │   └── db/
│   │       ├── index.ts         # Drizzle client init
│   │       ├── schema.ts        # All table definitions
│   │       └── migrations/      # Generated by drizzle-kit
│   │
│   ├── lib/                     # Shared utilities (used by app + worker)
│   │   ├── queues.ts            # BullMQ queue definitions
│   │   ├── llm.ts              # LLM client (OpenAI-compatible)
│   │   ├── minio.ts            # MinIO/S3 client
│   │   └── env.ts              # Typed env var access
│   │
│   └── styles/
│       └── globals.css          # Tailwind base + shadcn theme
│
├── worker/                      # Background job worker
    ├── Dockerfile               # Worker container build
    ├── index.ts                 # Entrypoint — registers all processors
    └── jobs/
        ├── process-mail.ts      # Parse + classify incoming email
        ├── extract-invoice.ts   # LLM-powered invoice data extraction
        ├── llm-task.ts          # Generic LLM prompt execution
        └── sync-agenda.ts       # CalDAV calendar sync

packages/
└── cli/                         # rik CLI
    ├── package.json
    ├── tsconfig.json
    ├── src/
    │   ├── index.ts             # Entry point, arg parsing
    │   ├── client.ts            # HTTP client for Rik chat API
    │   ├── interactive.ts       # REPL / interactive chat mode
    │   └── config.ts            # Read ~/.rikrc for endpoint URL
    └── bin/
        └── rik                  # CLI binary
```

## Config (`config/`)

```
config/
└── stalwart.toml               # Stalwart mail server configuration
```

## Data (`data/`)

Git-ignored. Created by Docker volumes on first run.

```
data/
├── rik.db                      # SQLite database file
├── minio/                      # MinIO object storage data
└── mail/                       # Stalwart mail storage
```

## Key Files by Purpose

| Task | File(s) |
|------|---------|
| Add a route/page | `app/src/routes/` |
| Add a server function | `app/src/server/functions/` |
| Modify the database schema | `app/src/server/db/schema.ts` |
| Add a background job | `app/worker/jobs/` + register in `app/worker/index.ts` |
| Add a UI component | `app/src/components/` (shadcn in `ui/`) |
| Chat / LLM tools | `app/src/server/functions/chat.ts` |
| CLI client | `packages/cli/src/` |
| Configure services | `config/`, `docker-compose.yml` |
| Change env vars | `.env.example`, `docker-compose.yml` |
| LLM prompts/config | `app/src/lib/llm.ts`, job files in `app/worker/jobs/` |
